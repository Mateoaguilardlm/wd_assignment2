3. JavaScript Validation Logic
JavaScript is used in the project to enhance user interaction and validate form data before submission. On the page recommend.html, users select their diet choices, budget range, and purpose of dining. Based on these choices, JavaScript will recommend a restaurant. The suggested restaurant is then automatically sent to reservation.html via URL. The register.html file contains a JavaScript program that checks all fields of the form. For example, validation of the username format, email format, phone number length, password strength, password confirmation, and selection of gender. This program displays error messages if any of the fields are invalid. In reservation.html, JavaScript is used to validate the required fields. It prevents users from choosing dates in the past. It also updates the deposit amount depending on which restaurant is chosen. In addition to that, the appropriate voucher or credit card fields are displayed depending on the chosen payment method. Furthermore, credit card number length is validated based on the card type. Lastly, if the user chooses the same option as email address, the reservation email is automatically copied in the billing email field.
